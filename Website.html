Discovery

Card Sorting Test

Card Sorting Test is a method uncoverring how the target audienceâ€™s domain knowledge is structured, and it serves to create an information architecture that matches usersâ€™ expectations. I invited 15 consumer to do this test to learn how users understand and interact the current content and CTA.


Steps
  • 1. Wrote 47 topics on the cards.
  • 2. Participants group cards into piles.
  • 3. Participants name each group.
  • 4. Participants explained their rationale why they grouped cards in that way.
  • 5. Use XSort tools to analyze the data collected.
  • 6. Reorganize the information based on the analysis.

Tree Testing and AB Testing

Tree testing helped me to know which option could allow audiences easier, faster to find the more accurate information without less back and forth clicks. I used Optimal Workshop to help me to measure the users actions.


Steps
  • 1. Structure the 47 topics into A/B options in the OPTIMAL WORKSHOP.
  • 2. List 5 contents for users to look for.
  • 3. Invite 10 participants to find the 5 content in each option.
  • 4. Analyze data to compare which works better.
